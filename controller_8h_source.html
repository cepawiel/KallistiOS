201 /** \brief Set an automatic button press callback.
202  \ingroup controller_inputs
203 
204  This function sets a callback function to be called when the specified
205  controller has the set of buttons given pressed.
206 
207  \note
208  The callback gets invoked for the given maple port; however, providing
209  an address of '0' will cause it to be invoked for any port with a
210  device pressing the given buttons. Since you are passed back the address
211  of this device, You are free to implement your own filtering logic within
212  your callback.
213 
214  \warning
215  The provided callback function is invoked within a context which has
216  interrupts disabled. This means that you should not do any sort of complex
217  processing or make any API calls which depend on interrupts to complete,
218  such as Maple or ethernet processing whichy rely on packet transmission,
219  any sleeping or threading calls, blocking on any sort of file I/O, etc.
220  This mechanism is typically used to quickly terminate the application
221  and should be used with caution.
222 
223  \param addr The controller to listen on (or 0 for all ports).
224  This value can be obtained by using maple_addr().
225  \param btns The buttons bitmask to match.
226  \param cb The callback to call when the buttons are pressed.
227 */

358 /** \defgroup controller_query_types Querying Types
359  \brief API for determining controller types
360  \ingroup controller
361 
362  The following API is for detecting between different types
363  of standard controllers. These controllers are not identified
364  by specific model but are instead identified solely by capabilities,
365  so that homebrew software can remain generic and future-proof to later
366  homebrew controllers or exotic, untested 3rd party peripherals.
367 
368  \warning
369  Usually you want to check if a controller <i>supports the
370  capabilities</i> of another controller, not whether it is has
371  the <i>exact</i> same capabilities of a controller. For example,
372  a controller that happens to come along supporting a dual analog
373  stick but is otherwise the same layout as a standard controller
374  would not match the standard controller type; however, it would
375  implement its capabilities. There exist 3rd party adapters for
376  connecting dual-analog PS2 controllers to DC which operate
377  like this today.
378 
379  \note
380  If you really want to hard-code the detection of a certain
381  exact model or brand of controller, instead of basing your
382  detection upon capabilities, check for its product_name
383  or license within the maple_devinfo structure.
384 
385  \sa cont_has_capabilities, maple_devinfo
386 */
